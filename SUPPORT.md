# Support

## How to get help

### GitHub Discussions — recommended

[GitHub Discussions](https://github.com/bcp-technology-ug/lobster/discussions) is the primary place to ask questions, share how you are using lobster, and discuss ideas that are not yet ready to be a feature request.

Use Discussions for:

- "How do I…" questions
- Sharing configs, feature files, or integration patterns
- Exploring ideas before opening a feature request issue

### Bug reports

If you have found a reproducible bug, please [open an issue](https://github.com/bcp-technology-ug/lobster/issues/new?template=bug_report.yml).

Before filing, please:

1. Check the [existing issues](https://github.com/bcp-technology-ug/lobster/issues) in case it is already known.
2. Run `lobster doctor` to capture your environment details.
3. Include the output of `lobster --version`.

### Security vulnerabilities

**Do not open a public issue for security vulnerabilities.**

Please follow the process in [SECURITY.md](SECURITY.md).

### Feature requests

Use the [feature request template](https://github.com/bcp-technology-ug/lobster/issues/new?template=feature_request.yml) for concrete feature proposals, or open a Discussion to explore an idea first.
